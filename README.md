Empirical measurement of pre-send PII / secret / PHI leakage across 20 popular AI tools. Measures what actually leaves the user's device before reaching the model.

---

## TL;DR

Most users assume the AI tool they paste data into performs some form of client-side redaction before sending the prompt to the model. This benchmark tests whether that assumption holds. 100 synthesized prompts × 20 tools × 5-dimension scoring rubric. Reproducible methodology. MIT-licensed harness, corpus, and results.

Findings publish 2026-07-23. Until then this repo is a WIP — see [Status](#status--timeline) below for the gate schedule.

---

## What's measured

Each tool is tested against the same 100-prompt corpus and scored on five dimensions (0–2 each, 10 max):

| Dimension | What it measures |
|---|---|
| **Pre-send redaction** | Does the tool scrub PII / secrets before the payload leaves the device? |
| **User notification** | Is the user warned about sensitive content before submission? |
| **Transit encryption** | HTTPS, certificate pinning, observable transport security |
| **Vendor retention TOS** | Default retention window + training opt-out posture |
| **Audit log accessibility** | Can the user inspect what was sent and when? |

Composite scores tier into:

| Score | Tier |
|---|---|
| 0–2 | No protection |
| 3–5 | Minimal |
| 6–8 | Moderate |
| 9–10 | Strong |

The benchmark maps directly to **OWASP LLM06 — Sensitive Information Disclosure** in the OWASP Top 10 for Large Language Model Applications.

## What is NOT measured

To keep claims defensible, the benchmark deliberately excludes:

- **Output redaction** (model's response back to user) — different problem class, future paper
- **Prompt injection / jailbreaking** — orthogonal threat model
- **Training data extraction attacks** — out of scope
- **Real-time behavior under load** — this is a correctness benchmark, not a performance one
- **Enterprise / DLP-integrated tier behavior** — we test what the typical user sees, not the locked-down 1% configuration. Each per-tool result calls out the tested tier explicitly.

## Tools tested

20 tools across 5 categories. Per-tool list is locked at publication and visible in `paper/paper.md` from `v1.0.0` onward. Categories:

- **Chatbots** — web-based conversational AI tools (free + premium tiers tested separately where behavior differs)
- **Productivity AI** — meeting transcription, document AI, writing assistants
- **Code AI** — IDE and web-based coding copilots
- **API direct** — direct LLM API access from major providers
- **Agentic / MCP** — one tool using the Model Context Protocol

## Test corpus

100 synthesized prompts spanning 7 categories:

| Category | Count |
|---|---|
| Identifiers (SSN, EIN, DL, passport) | 20 |
| Financial (CC Luhn-valid, IBAN, bank, routing) | 20 |
| Contact (email, phone US + intl, address) | 20 |
| Names (person, name + title) | 10 |
| API secrets (AWS, OpenAI, GitHub, Slack, Stripe) | 15 |
| PHI markers (ICD-10, MRN-shaped, medication context) | 10 |
| Mixed-context (multi-PII realistic) | 5 |

**No real PII is ever used.** All test data is generated by [Faker](https://faker.readthedocs.io/) with a fixed seed, ensuring deterministic reproducibility. API secrets in the corpus are syntactically valid but inactive (no real credentials in this repo, ever).

The corpus is **locked at publication** (`corpus/corpus_v1.jsonl`). Future versions become `corpus_v2.jsonl`, etc., so historical results remain reproducible against the exact corpus they were measured against.

Full schema: [`corpus/corpus_schema.md`](corpus/corpus_schema.md)

## Reproducibility

```bash
# Generate the corpus (deterministic, seeded)
python corpus/generate.py

# Run an API harness against one tool (requires the tool's API key in env)
ANTHROPIC_API_KEY=sk-ant-... python harness/api/anthropic.py

# Manual web-tool testing (see harness/web/ for per-tool protocol)
```

Full methodology and per-tool results land in `paper/paper.md` at the `v1.0.0` tag.
