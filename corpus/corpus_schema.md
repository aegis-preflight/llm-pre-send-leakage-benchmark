# Corpus Schema

The test corpus is stored as JSONL — one JSON object per line, one object per prompt. Locked at publication as `corpus_v1.jsonl` and never modified post-release.

## Why JSONL

- Line-addressable (each prompt is independently inspectable)
- Diff-friendly in git (changes show per-prompt, not as whole-file rewrites)
- Streamable for large corpora (future v2+ may exceed 100 prompts)

## Per-prompt fields

| Field | Type | Required | Description |
|---|---|---|---|
| `id` | string | yes | Stable identifier. Format: `<category-prefix>-<3-digit-index>`. Examples: `ident-001`, `fin-012`, `secret-005`. Once assigned, never reused. |
| `category` | string | yes | One of: `identifiers`, `financial`, `contact`, `names`, `api_secrets`, `phi`, `mixed`. |
| `subcategory` | string | yes | More specific type within the category. E.g., `ssn`, `credit_card`, `aws_access_key`, `icd10_code`. |
| `prompt_text` | string | yes | The exact text submitted to each tool under test. Must read like a plausible user prompt (i.e., natural-language wrapping around the PII), not a raw PII dump. |
| `expected_categories` | array<string> | yes | The semantic ground-truth — what categories of sensitive data the prompt was *designed* to contain. E.g., `["SSN", "EMAIL"]`. Used as the comparison target during scoring. |
| `expected_count` | object | yes | Map of `<category>` → integer expected count. E.g., `{"SSN": 1, "EMAIL": 1}`. Lets the scorer detect partial leaks (some categories scrubbed, others not). |
| `aegis_detect` | array<object> | yes | What the ground-truth detector finds when run against `prompt_text` (before any tool processes it). Each object is `{"type": "<DetectionType>", "count": <int>}`. Used to sanity-check that the prompt actually contains what `expected_categories` claims. |
| `notes` | string | no | Free-form notes about the prompt (e.g., why a particular wording was chosen, edge-case context). |

## Example record

```json
{
  "id": "ident-001",
  "category": "identifiers",
  "subcategory": "ssn",
  "prompt_text": "Please verify customer record: SSN 123-45-6789.",
  "expected_categories": ["SSN"],
  "expected_count": {"SSN": 1},
  "aegis_detect": [{"type": "SSN", "count": 1}],
  "notes": ""
}
```

## Generation rules

1. **No real PII.** Every PII-shaped value is generated by [Faker](https://faker.readthedocs.io/) with a fixed seed declared in `generate.py`. Re-running `generate.py` with the same seed produces byte-identical output.
2. **API secrets are syntactically valid but inactive.** Generated to match each provider's documented format (e.g., AWS access key = `AKIA` + 16 base32 chars) without ever matching a real, live credential.
3. **PHI keywords are reference codes, not patient data.** ICD-10 codes (e.g., `E11.9`, `I10`) are public medical reference codes. MRN-shaped IDs are synthetic.
4. **Names** use Faker's locale-aware generators across at least 3 locales (`en_US`, `en_GB`, `de_DE`) for variety.
5. **Prompt wrapping is realistic.** Every prompt reads like a plausible end-user paste — "Please summarize this customer note...", "Can you debug why this API key isn't working...", "Draft an email to John about his account..." — not a synthetic stress-test dump.

## Versioning

- `corpus_v1.jsonl` is locked at the `v1.0.0` tag (paper publication 2026-07-23).
- Any future changes ship as `corpus_v2.jsonl` (etc.) — the v1 file is never edited after lock.
- This guarantees that benchmark results published against v1 remain reproducible against the exact corpus that was used.

## Reproducing the corpus

```bash
python corpus/generate.py
diff <(jq -S . corpus/corpus_v1.jsonl) <(jq -S . corpus/corpus_v1.expected.jsonl)
# (Should produce empty diff after lock — corpus is byte-deterministic.)
```

## Ground-truth detector

The `aegis_detect` field is populated by running the [aegis-core](https://aegispreflight.com) detection library against `prompt_text`. The library implements regex-based detection plus Luhn (credit card), IBAN checksum, and SSN structural validation. The detection patterns are documented in the published paper. As an open-source cross-check, [Microsoft Presidio](https://microsoft.github.io/presidio/) is run against a ten percent sample of the corpus and the results compared.
